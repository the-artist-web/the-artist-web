موقع git

Git هو نظام للتحكم في الإصدار الموزع (DVCS) مصمم لتتبع التغييرات في كود المصدر أثناء تطوير البرامج. تم إنشاؤه بواسطة Linus Torvalds في عام 2005 وأصبح أحد أنظمة التحكم في الإصدار الأكثر استخدامًا في العالم. Git هو برنامج مجاني ومفتوح المصدر، يوفر حلاً قويًا ومرنًا لإدارة مشاريع البرامج التعاونية. فيما يلي بعض المفاهيم والميزات الأساسية لـ Git: التحكم في الإصدار: يسمح Git للمطورين بتتبع التغييرات في كود المصدر الخاص بهم متأخر , بعد فوات الوقت. يتم تسجيل كل تغيير في التزام، ويتم تخزين سجل التغييرات بالكامل في مستودع Git. وزعت النظام: Git هو نظام للتحكم في الإصدار الموزع، أي أن كل مطور لديه نسخة كاملة من المستودع بأكمله، بما في ذلك تاريخها، على أجهزتهم المحلية. تسمح هذه اللامركزية بالعمل دون اتصال بالإنترنت والوصول بشكل أسرع إلى سجل الإصدارات. التفرع والدمج: يدعم Git التفرع، مما يسمح للمطورين بإنشاء فروع منفصلة لميزات مختلفة أو إصلاحات للأخطاء. يمكن دمج الفروع مرة أخرى في قاعدة التعليمات البرمجية الرئيسية، مما يسهل التطوير والتجريب الموازي. مثال: نسخ كود باش # إنشاء فرع جديد git Branch feature-branch # التبديل إلى الفرع الجديد git checkout feature-branch # إجراء التغييرات والالتزام و ثم ادمج مرة أخرى git checkout main git merge feature-branch إجراء التغييرات: يستخدم المطورون أمر git Commit لحفظ التغييرات إلى مستودعهم المحلي. يحتوي كل التزام على معرف فريد (تجزئة) ويتضمن رسالة تصف التغييرات التي تم إجراؤها. مثال: bash انسخ الكود # مرحلة التغييرات git add . # تنفيذ التغييرات برسالة git الالتزام -m "الميزة المنفذة X" المستودعات البعيدة: يتيح Git للمطورين مزامنة مستودعاتهم المحلية مع المستودعات البعيدة المستضافة على منصات مثل GitHub أو GitLab أو بيتبوكيت. يتيح ذلك التعاون بين أعضاء الفريق ويوفر موقعًا مركزيًا للمشروع. مثال: نسخ كود باش # إضافة مستودع بعيد git Remote add Origin <repository_url> # دفع التغييرات إلى المستودع البعيد git Push -u Origin main طلبات السحب: على منصات مثل GitHub، يمكن للمطورين اقتراح التغييرات عن طريق إنشاء طلبات السحب. طلب السحب يسمح للفريق يقوم الأعضاء بمراجعة ومناقشة التغييرات المقترحة قبل دمجها في الفرع الرئيسي. وضع العلامات: يسمح Git للمطورين بالإنشاء علامات لوضع علامة على نقاط محددة في سجل الإصدارات، مثل نقاط الإصدار أو الأحداث الهامة. مثال: نسخ كود باش # إنشاء علامة توضيحية git tag -a v1.0 -m "Release version 1.0" # دفع العلامات إلى المستودع البعيد git Push --tags Conflict Resolution: عندما تتعارض التغييرات التي أجراها مطورون مختلفون، يوفر Git أدوات لحل التعارضات يدويًا. يمكن للمطورين اختيار كيفية القيام بذلك دمج التغييرات المتعارضة. يستخدم Git على نطاق واسع في صناعة تطوير البرمجيات ويعتبر أداة أساسية للإصدار السيطرة والتعاون. وهو مدعوم على أنظمة تشغيل مختلفة، بما في ذلك Linux وmacOS وWindows.

مطور الواجهة الأمامية

عرض الكل

git config --global user.name " "

بعرف الجهاز باسمي

git config --global user.email " "

بعرف الجهاذ بي ايميلي

git config user.name

بجيب بيها الاسم بتاعي الي سجلت بيه

git config user.email

بتجيب الايميل بتاعي الي سجلت بيه

git bash

اسم الصفحه الي بينكتب فيها الاوامر

git

بتجيب كل حاجه اقدر استخدمها في the git

clear

بتحذف اي حاجه مكتوبه في ال git

git init "myopject"

بيعملي مستودع و بيعمل ملف مخفي اسمه .git

cd myproject

بفتح بييها المستودع الي انشاته

ls -a

بيظهرلي ملف ال git المخفي

git status

بيشوفلي فيكام كومنت و كام سطر و مهمه جدا

touch index.html

بعمل بيها ملف index.html علشان اضيفه لي ملف ال myproject الي عملته

git add index.html

بدخل بيها ملف ال index.html الي ضيفته بي ال touch لي ملف ال myopject المخفي

git add index.html main.js

بضيف بيها كذا ملف مش ملف واحد

git commit -m"creat index.html"

بضيف بيها الملفات علي المسرح

git log

بيجيبلي ال commit الي ضيفتها

git branch

بعمل بيها ال branch قبل ما ادخل عليه بي ال git checkout

git checkout

بستخدمها في اني ارجع اي حاجه عملتها غلط عن طريق الكود الي موجود في ال commit و بعمل pranch اشتغل عليه بعيد عن المشروع الاصلي

git merge

بدمج النسخه الي هشتغل عليها مع الملف الاصلي

git remote -v

بشوف هل في repo علي الجهاز او لا الي هوا دمج بين git and github

git remote add origin

بضيف بيها ال repo الي بيربط بين ال git and github

git branch -m main

بيحول اسم البرانس من master الي main

git push -u origin master

بضيف بيها اي تعديلات حصلت علي الملفات للملفات علي ال github
